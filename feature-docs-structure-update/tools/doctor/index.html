    Doctor¶

    The DrevOps Doctor is a standalone, self-contained script designed to inspect the current state of your DrevOps project. Its primary functions include checking project requirements and displaying system information.

    Also, it runs before and after you build the project with ahoy build to make sure that all the requirements are met.

    Features¶

    The DrevOps Doctor script performs a series of checks to ensure the project environment is correctly set up:

    • Availability of the necessary tools (Docker, Docker Compose, Pygmy, Ahoy etc.)
    • Port availability on the host machine
    • Pygmy availability
    • Container status
    • Presence of SSH key within a container
    • Webserver status
    • Application bootstrap status
